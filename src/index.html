Setup & Config

git init [directory] Create a new local repository
git clone <url> [folder] Clone a remote repository locally
git config --global user.name "Name" Set commit author name globally
git config --global user.email "e@x" Set commit author email globally
git config --local user.email "work@co" Override email for this repo only
git config --list --show-origin Show all config values and where they're set

Example — New Project from Scratch

# Create and initialise a new project
mkdir my-app && cd my-app
git init
git config user.name "Alice Smith"
git config user.email "alice@example.com"

# Clone via HTTPS
git clone https://github.com/user/repo.git

# Clone via SSH (preferred when you have push access)
git clone git@github.com:user/repo.git

# Clone into a specific folder name
git clone git@github.com:user/repo.git myproject

Staging & Commits

git status Show working tree and staging area status
git add <file> Stage a specific file
git add . Stage all changes in current directory
git add -p Interactively select hunks to stage
git commit -m "message" Commit staged changes with a message
git commit -am "message" Stage all tracked changes and commit in one step
git commit --amend Edit the last commit message or add staged files
git diff Show unstaged changes
git diff --staged Show changes already staged for next commit

Example — Partial Staging & Amend

# Stage only specific hunks from a file
git add -p src/auth.js
# Press y/n to stage/skip, s to split hunk, q to quit

# Commit and then spot a typo in the message
git commit -m "fix authentcation bug"
git commit --amend -m "fix authentication bug"

# Forgot to include a file? Add it to the last commit
git add src/forgot-this.js
git commit --amend --no-edit

Branching & Merging

git branch List local branches (* marks current)
git branch -a List all branches including remotes
git branch <name> Create a new branch at current commit
git branch -d <name> Delete a fully merged branch
git branch -m <old> <new> Rename a branch
git switch <branch> Switch to an existing branch
git switch -c <new-branch> Create and switch to a new branch
git merge <branch> Merge branch into current (fast-forward if possible)
git merge --no-ff <branch> Always create a merge commit (preserves branch history)
git rebase main Replay current branch commits on top of main
git rebase -i HEAD~3 Interactive rebase — squash, reword, or drop last 3 commits

Example — Feature Branch Workflow

# Start from an up-to-date main
git switch main && git pull
git switch -c feature/user-auth

# After a few rough commits, clean them up interactively
git commit -m "wip: login form"
git commit -m "wip: hook up API"
git rebase -i HEAD~2
# In editor: change second "pick" to "squash", save & write message

# Rebase onto latest main before requesting a merge
git fetch origin
git rebase origin/main

# Merge into main with an explicit merge commit
git switch main
git merge --no-ff feature/user-auth

Remote & Sync

git remote -v List remotes with their fetch/push URLs
git remote add <name> <url> Register a new remote
git remote rename <old> <new> Rename a remote
git remote remove <name> Remove a remote
git fetch [remote] Download objects and refs without merging
git pull Fetch and merge (or rebase) remote changes
git pull --rebase Fetch and rebase your commits on top
git push -u origin <branch> Push and set upstream tracking branch
git push --force-with-lease Safe force-push — aborts if remote has new commits

Example — Forking Workflow

# Clone your fork, then add the original as "upstream"
git clone git@github.com:you/repo.git && cd repo
git remote add upstream git@github.com:original/repo.git

# Sync your local main with upstream
git fetch upstream
git switch main
git rebase upstream/main
git push origin main

# Create a feature branch, push, and set tracking in one go
git switch -c fix/typos
git commit -am "fix: correct typos in README"
git push -u origin fix/typos

Stash & Clean

git stash Shelve uncommitted changes (tracked files)
git stash push -u -m "message" Stash with a label, including untracked files
git stash list Show all stashed changesets
git stash pop Restore most recent stash and remove it
git stash apply stash@{2} Apply a specific stash without removing it
git stash show -p stash@{0} Show the diff stored in a stash
git stash drop stash@{0} Delete a specific stash entry
git clean -n Dry run: preview what would be deleted
git clean -fd Remove untracked files and directories

Example — Stash to Switch Contexts

# Mid-feature work, urgent bug arrives
git stash push -m "wip: checkout flow redesign"

# Fix bug on a dedicated branch
git switch main
git switch -c hotfix/payment-crash
git commit -am "fix: prevent nil pointer in payment handler"
git push -u origin hotfix/payment-crash

# Return to the feature
git switch feature/checkout
git stash pop

# Nuke untracked build artifacts (dry run first)
git clean -n
git clean -fd

History & Inspection

git log --oneline --graph --all Compact visual graph of all branches
git log --author="Alice" --since="2w" Filter commits by author and time window
git log --grep="fix" -p Search commits by message and show their diffs
git log -S "functionName" Find commits that added/removed a string (pickaxe)
git blame <file> Show who last changed each line of a file
git show <commit> Show a commit's metadata and diff
git reflog Full history of HEAD movements — essential for recovery
git bisect start Begin binary search for a regression
git bisect good <hash> / bad Mark commits; git checks out the midpoint each round
git bisect reset End bisect session and return to original HEAD

Example — Log Graph & Bisect to Find a Bug

# Pretty graph of all branches and tags
git log --oneline --graph --all --decorate

# Binary search: tests failing since some recent commit
git bisect start
git bisect bad
# Point to a commit you know was good
git bisect good v2.1.0
# Git checks out the midpoint — run your tests
npm test
git bisect good
# Repeat until git names the first bad commit
git bisect bad
git bisect reset

Undoing Things

git restore <file> Discard working-tree changes to a file
git restore --staged <file> Unstage a file (working tree untouched)
git reset --soft HEAD~1 Undo last commit — changes stay staged
git reset --mixed HEAD~1 Undo last commit — changes go back to unstaged (default)
git reset --hard HEAD~1 Undo last commit — changes are discarded entirely ⚠
git revert <commit> New commit that inverts a change (safe on shared branches)
git revert HEAD~3..HEAD Revert a range of commits

Example — Committed to the Wrong Branch

# Accidentally committed feature work directly to main
git log --oneline -4

# Carry those commits onto a new branch, then reset main
git switch -c feature/accidental-work
git switch main
git reset --hard origin/main

# Alternative: undo commit, keep changes, re-commit on right branch
git reset --soft HEAD~1
git stash
git switch -c feature/my-work
git stash pop
git commit -m "feat: my work on correct branch"

# Revert a bad commit that is already pushed to shared branch
git revert abc1234
git push

Tags & Releases

git tag List all tags (alphabetically)
git tag v1.0.0 Lightweight tag at current commit (just a pointer)
git tag -a v1.0.0 -m "Release" Annotated tag with message, tagger, and date (preferred)
git tag -a v1.0.0 <commit> Tag a specific past commit
git show v1.0.0 Show tag metadata and the tagged commit
git push origin v1.0.0 Push a single tag to remote
git push origin --tags Push all local tags to remote
git tag -d v1.0.0 Delete a local tag
git push origin :refs/tags/v1.0.0 Delete a remote tag

Example — Tagging a Release

# Make sure you're on the right commit
git switch main && git pull

# Create an annotated tag (stores tagger, date, message)
git tag -a v1.0.0 -m "Release v1.0.0: initial stable release"

# Push the tag — can trigger a GitHub Actions release workflow
git push origin v1.0.0

# Oops, tagged the wrong commit — delete and re-create
git tag -d v1.0.0
git push origin :refs/tags/v1.0.0
git tag -a v1.0.0 HEAD -m "Release v1.0.0: initial stable release"
git push origin v1.0.0

GitHub CLI (gh)

Auth & Repo

gh auth login Authenticate with GitHub (browser or token)
gh repo create <name> --public Create a new GitHub repository
gh repo clone <owner/repo> Clone a repo using SSH if authenticated
gh repo fork <owner/repo> --clone Fork and clone in one step
gh repo view --web Open current repo in browser

Issues

gh issue create -t "Title" -b "Body" Create a new issue
gh issue list --assignee @me List issues assigned to you
gh issue close 42 -c "resolved in #99" Close issue #42 with a comment
gh issue view 42 --web Open issue in browser

Pull Requests

gh pr create --fill Create PR using branch name & commit messages as body
gh pr list List open pull requests in current repo
gh pr checkout 99 Fetch and check out PR #99 locally
gh pr merge 99 --squash --delete-branch Squash-merge PR and remove the branch
gh pr view --web Open current branch's PR in browser

Runs & Gists

gh run list --limit 5 Show last 5 Actions runs
gh run watch Stream live output of the latest run
gh run view --log-failed Show only the logs from failed steps
gh gist create notes.md --public Create a public gist from a file

Example — Full PR Workflow with gh

# Start from an issue
gh issue view 42

# Create branch, do work, push with upstream tracking
git switch -c feat/dark-mode
git commit -am "feat: add dark mode toggle"
git push -u origin feat/dark-mode

# Open a PR that closes the issue, requesting reviewers
gh pr create --title "feat: dark mode toggle" \
  --body "Closes #42" \
  --reviewer alice,bob

# Watch CI live while you wait for review
gh run watch

# Once approved: squash-merge and clean up the branch
gh pr merge --squash --delete-branch

.gitconfig Tips

git config --global alias.co checkout Alias: git co → git checkout
git config --global pull.rebase true Always rebase instead of merge when pulling
git config --global init.defaultBranch main New repos start on main instead of master
git config --global push.autoSetupRemote true git push auto-sets upstream — no more -u
git config --global core.editor "code --wait" Use VS Code for commit messages and interactive rebase
git config --global core.pager delta Use delta for syntax-highlighted diffs (install separately)

Recommended ~/.gitconfig

[user]
    name = Alice Smith
    email = alice@example.com

[init]
    defaultBranch = main

[pull]
    rebase = true

[push]
    autoSetupRemote = true

[core]
    editor = code --wait
    pager = delta

[delta]
    navigate = true
    line-numbers = true
    side-by-side = true

[alias]
    co = checkout
    br = branch
    st = status
    lg = log --oneline --graph --all --decorate
    last = log -1 HEAD --stat
    unstage = restore --staged

[credential]
    helper = osxkeychain

GitHub Actions Basics

Key Concepts

on: push / pull_request Trigger on code push or PR open/update
on: schedule: cron: '0 9 * * 1' Cron trigger (every Monday at 09:00 UTC)
on: workflow_dispatch Allow manual trigger from the GitHub UI
runs-on: ubuntu-latest Run job on a GitHub-hosted Ubuntu runner
uses: actions/checkout@v4 Check out the repository code
uses: actions/setup-node@v4 Install and configure a Node.js version
uses: actions/cache@v4 Cache dependencies to speed up runs
${{ secrets.MY_SECRET }} Reference a repository or org secret
${{ github.sha }} / github.ref_name Access GitHub context: SHA, branch name, actor, etc.

Matrix Build

strategy:
  fail-fast: false
  matrix:
    node: [18, 20, 22]
    os: [ubuntu-latest, windows-latest]

runs-on: ${{ matrix.os }}
# Spawns 6 parallel jobs (3 node versions × 2 OSes)
steps:
  - uses: actions/setup-node@v4
    with:
      node-version: ${{ matrix.node }}

Dependency Caching

- uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-

Example — CI Workflow: Test on Push to main & on All PRs

name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build
        env:
          API_KEY: ${{ secrets.API_KEY }}
